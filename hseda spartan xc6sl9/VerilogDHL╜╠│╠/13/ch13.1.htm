13.1 Specify Block Declaration

A block statement called the specify block is the vehicle for describing paths between a source and a destination and assigning delays to these paths. The syntax for specify block is shown below:

Syntax 13-1 : Syntax of specify block

The specify block shall be bounded by the keywords specify and endspecify , and shall appear inside a module declaration. The specify block can be used to perform the following tasks:

  • Describe various paths across the module.
  • Assign delays to those paths.
  • Perform timing checks to ensure that events occurring at the module inputs satisfy the timing constraints of the device described by the module. See .

The paths described in the specify block, called module paths , pair a signal source with a signal destination. The source may be unidirectional (an input port) or bidirectional (an inout port) and is referred to as the module path s ource . Similarly, the destination may be unidirectional (an output port) or bidirectional (an inout port) and is referred to as the module path desti nation .

The first two lines following the keyword specify declare specify parameters, which are discussed in See Declaring parameters in specify blocks . The line following the declarations of specify parameters describes a module path and assigns delays to that module path. The specify parameters determine the delay assigned to the module path. Specifying module paths is presented in See Module path declarations . Assigning delays to module paths is discussed in See Assigning delays to module paths . The line preceding the keyword endspecify instantiates one of the system timing checks, which are discussed further in .
