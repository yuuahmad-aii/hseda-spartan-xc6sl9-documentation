13.6 Driving wired logic

Module path output nets shall not have more than one driver within the module. Therefore, wired logic is not allowed at module path outputs.

See : Illegal module paths illustrates a violation of this wired-output rule, and a method of avoiding the rule violation.

Figure 13-5: Legal and illegal module paths

In See : Legal and illegal module paths (a), any module path to S is illegal because the path destination has two drivers.

Assuming signal S in See : Legal and illegal module paths (a) is a wired-and , this limitation can be circumvented by replacing wired logic with gated logic to create a single driver to the output. See : Legal and illegal module paths (b) shows how adding a third and gate--the shaded one--solves the problem for the module in See : Legal and illegal module paths (a).

The example in See : Illegal module paths is also illegal. In this example, when the outputs Q and R are wired together, it creates a condition where both paths have multiple drivers from within the same module.

Figure 13-6: Illegal module paths


Although multiple output drivers to a path destination are prohibited inside the same module, they are allowed outside the module. The example in See : Legal module paths is legal since Q and R each have only one driver within the module in which the module paths are specified.
