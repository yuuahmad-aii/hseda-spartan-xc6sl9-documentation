3.5   Implicit declarations

The syntax shown in 3.2 shall be used to explicitly declare nets and registers. In the absence of an explicit declaration of a net or a register, statements for gate, user-defined primitive, and module instantiations shall assume an implicit net declaration. This happens when a name is used in the terminal list of an instance of a gate, a user-defined primitive, or a module that has not been explicitly declared previously in one of the declaration statements of the instantiating module. See 7.9.

These implicitly declared nets shall be treated as scalar nets of type wire. See Section 16 for a discussion of control of the type for implicitly declared nets with the `default_nettype compiler directive.
